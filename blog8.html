Creation of EC2 Instances – part 6

EC2 instances are a major part of this infrastructure. There will be a total of 4 instances. Two public and two private. The public instances will be located in the public subnets and the private instances will be located in private subnets.

EC2 Creation via Terraform

The resource “aws_instance” will be used to create an EC2 instance. "private1" will be the variable name. The ami is "ami-042e8287309f5df03". The ami identifier can be found on the aws console. Navigate to console.aws.amazon.com > EC2 > Lunch Instance to see all the available EC2s. The instance type which is “instance_type = "t2.micro" ” will also be found there. “subnet_id = aws_subnet. public_subnet1.id” will be used to attach the instance to the correct subnet. In this case, the EC2 instance will be attached to “public_subnet1”. This subnet was created in blog 4. Next, security_groups = [aws_security_group. main_security_group.id] will be used to attach a security group to the instance. In order to see how “main_security_group” was created, visit blog 7. Finally, declare the correct availability zone by using “availability_zone = "us-east-1c" ”. Tags are optional.

Public instance #1

resource "aws_instance" "public1" {
ami = "ami-042e8287309f5df03"
instance_type = "t2.micro"
subnet_id = aws_subnet. public_subnet1.id
security_groups = [aws_security_group.main_security_group.id]
availability_zone = "us-east-1a"

  tags = {
    Name = "public1"
  }
}

Repeated the same steps for the following instance with a different variable name, tag name, subnets, and availability zones.
