Creating the Elastic Load Balancer

In todayâ€™s blog, I will be creating the elastic load balancer for the AWS infrastructure. Elastic Load Balancing allows users to monitor the health of their applications and their performance in real time with Amazon CloudWatch metrics, logging, and request tracing. It also allows traffic to be sent to different EC2 instances, in different availability zones. The reason for that is to make sure the website stays live, in case something goes wrong with one of the EC2 instances where the website is being hosted from.

ELB Terraform

resource "aws_elb" "runcmd" {

name = "elb"
security_groups = [aws_security_group.first.id]
subnets = [aws_subnet.ngw.id, aws_subnet.ngw2.id]
listener {
instance_port = 80
instance_protocol = "http"
lb_port = 80
lb_protocol
= "http"
}

health_check {
healthy_threshold = 2
unhealthy_threshold = 2
timeout = 3
target = "HTTP:80/"
interval = 30
}

instances = [aws_instance.pri_1.id, aws_instance.pri_2.id]
cross_zone_load_balancing = true
idle_timeout = 400
connection_draining = true
connection_draining_timeout = 400

tags = {
Name = "new_elb"
}
}
