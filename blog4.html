AWS Infrastructure Creation via Terraform for CIT Design project – Part 2 (Public & Privet Subnets)

In today’s blog, I will continue with the part two of the “AWS Infrastructure Creation via Terraform for CIT Design project” series. In this step, I will demonstrate how to create two public and two private subnets in two different availability zones using Terraform.


Step 1: Creating the public subnets
In order to create subnets resource "aws_subnet" is going to be used. I will name the variable "public_subnet1" . Next, the cidr block will be indicated using cidr_block = "10.0.1.0/24" This subnet will be created with the VPC which I created in blog3 using the following line of code vpc_id = aws_vpc.main_VPC.id . Next, map_public_ip_on_launch = true will Indicate whether instances launched in this subnet receive a public IPv4 address. availability_zone = "us-west-2a" will specify its availability zone.

resource "aws_subnet" "public_subnet1" {

cidr_block = "10.0.1.0/24"
vpc_id = aws_vpc.main_VPC.id
map_public_ip_on_launch = true
availability_zone = "us-west-2a"

tags = {
Name = "public_subnet_1"
      }
}


The same lines of code will be used to create the second public subnet; however, it will have different variable name, tag name, and availability zone.

resource "aws_subnet" "private_subnet1" {

cidr_block = "10.0.3.0/24"
vpc_id = aws_vpc.main_VPC.id
map_public_ip_on_launch = true
availability_zone = "us-west-2a"
tags = {
Name = "private_subnet_1"
      }
}
